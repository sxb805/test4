# 客户信息管理 PRD（轻量版）

## 1. 模块信息
- 模块名称：客户信息管理
- 使用角色：后台管理员

## 2. 功能概述
- 支持客户信息全生命周期管理：新增、编辑、删除、批量删除、查询。
- 支持分页展示与多条件筛选，提升日常运营效率。
- 支持顶部数据统计卡片，快速查看客户状态分布。
- 支持客户状态彩色标签展示，增强信息可读性。

## 3. 核心字段
- 客户名称
- 客户编号
- 联系人
- 联系电话
- 所属行业
- 客户状态
- 所在地区
- 备注
- 创建时间
- 更新时间

## 4. 页面说明

### 4.1 页面结构
1. 顶部统计区（4 个卡片）
2. 条件筛选区
3. 操作区（新增、批量删除）
4. 列表区（分页表格）

### 4.2 顶部统计卡片
- 总客户数
- 正常客户
- 意向客户
- 停用客户

### 4.3 筛选条件
- 客户名称（模糊匹配）
- 所属行业（下拉）
- 客户状态（下拉）
- 创建时间范围（开始时间、结束时间）

### 4.4 列表展示字段
- 客户名称
- 客户编号
- 联系人
- 联系电话
- 所属行业
- 客户状态（彩色标签）
- 所在地区
- 备注
- 创建时间
- 更新时间
- 操作（编辑、删除、详情）

### 4.5 页面交互功能
- 新增客户
- 编辑客户
- 单条删除
- 批量删除
- 分页浏览
- 详情查看

## 5. 业务规则
- 客户编号全局唯一。
- 客户名称为必填项，建议长度 2~100。
- 联系电话需通过格式校验（手机号/固话）。
- 客户状态枚举：
  - `NORMAL`：正常
  - `INTENTION`：意向
  - `DISABLED`：停用
- 删除策略建议采用逻辑删除（保留审计追踪能力）。
- 批量删除仅处理存在且未删除的数据，返回成功/失败数量。
- 创建时间、更新时间由系统自动维护。
- 列表默认按更新时间倒序排序。

## 6. UI 设计要求
- 风格：商务简约风。
- 布局：分层清晰，避免纯裸表格。
- 视觉：卡片阴影、舒适留白、统一排版节奏。
- 强调：统计信息与业务状态可视化（状态标签颜色区分）。
