# 后端 CRUD + 导入导出提示词模板（最佳实践版）

@skill my-crud-skill-common

默认技能：`my-crud-skill-common`（普通单表 Spring Boot + MyBatis-Plus CRUD，不含分表）。
若业务明确需要分表（month/year/hash），再切换到 `my-crud-skill-sharding`。

请基于现有项目规范与 example 参考模块，生成【{title}】模块后端完整 CRUD + 导入导出代码。

【生成目标】
- 模块中文名：`{title}`
- 资源名：`{resource}`
- 交付标准：`{deliveryLevel}`（可选：`最低交付` / `标准交付` / `严格交付`）
- 表字段：
  - `{fields}`

【必须实现】
1. 分页查询
2. 列表查询
3. 根据ID查询
4. 新增
5. 修改
6. 删除
7. 导入
8. 导出
9. 必要校验（唯一性、必填、关联有效性）

【强制业务规则】
1. 唯一字段校验（新增/修改均校验，修改需排除自身）
2. 关联字段遵循“关联ID + 冗余名称”模式（如 `exampleId + exampleName`）
3. 人员字段遵循“ID校验 + 名称回填”模式（如 `staffId -> umsService.getUserByStaffId -> 回填staffName`）
4. 所有业务失败必须返回可读 `msg`

【导入导出口径（默认）】
1. 导入模板与导出列不包含 `id`
2. 导入模板与导出列不包含关联 `id`（如 `exampleId`、`staffId`）
3. 导入使用可读字段（如 `exampleName`、`staffName`）
4. 导入时根据可读字段反查并回填关联ID
5. 导出以业务可读字段为主

【实现约束】
1. 参考 example 模块的分层结构、返回结构、异常处理风格
2. 禁止改动 example 的整体架构，只替换资源与业务字段
3. QueryDTO 字段在查询构造中逐项落地，禁止漏条件
4. 分页接口兼容 `Pageable`（`page` 0基、`size`）
5. 导入实现采用“批量预取 + 前置校验 + 批量落库”，禁止循环内远程调用
6. 写操作使用事务（按项目规范）

【交付说明】
1. 变更文件清单
2. 接口清单（路径+入参+出参）
3. 导入字段映射与校验逻辑说明
4. 测试执行结果（按 `{deliveryLevel}` 执行）
5. 若为“严格交付”，必须输出 `docs/reports/` 下可追溯测试报告路径

【测试与门禁补充（强制）】
1. 生成业务代码时必须同步生成测试代码（单元/接口），禁止后补。
2. 字段口径测试需覆盖：必填、长度、枚举、格式、唯一性、关联字段回填。
3. 复杂规则需覆盖：跨字段约束、状态流转一致性。
4. 覆盖率门禁默认采用：
   - 后端总体：`line >= 70%`、`branch >= 60%`
   - 本次改动核心模块：`line >= 80%`
5. 门禁触发时机：分支合并前必须通过，不允许合并后补测。
